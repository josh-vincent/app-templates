# Fates, Capabilities & Archetypes

> Generated by `node scripts/catalog/build-index.mjs` from `fates.json` — edit that file, not this one.

A **fate** is what the app is destined to be (spike → mvp → beta → store-ready);
a **capability** is a composable feature (ota-updates, local-first, no-auth, …);
an **archetype** is the app's shape (game, directory, marketplace, …).
Declare them in a remix plan or as `remix.mjs suggest` flags; audit with `check.mjs`.

## Fates

### `spike` — Spike — prove the idea

One happy path, hardcoded mock data, no auth, no error handling. Throwaway by design; exists to feel the core loop on a device.

**Requires screen categories:** home

**Checklist:**

- [ ] Runs in Expo Go / dev client *(manual)*
- [ ] The one core-loop screen works end to end with mock data *(manual)*

### `mvp` — MVP — first real users

The full skeleton: navigation shell, auth decision made (auth, no-auth, or demo-mode), loading/empty/error states everywhere a user can reach, settings, and the core feature set.

**Requires capabilities:** `skeleton-shell`

**Requires screen categories:** home, settings, profile

**Checklist:**

- [ ] Auth strategy decided and implemented (auth screens, no-auth, or demo-mode)
- [ ] 404 catch-all, error boundary, and empty states present
- [ ] Light and dark mode both render correctly
- [ ] Settings screen exists
- [ ] npx tsc --noEmit passes *(manual)*

### `beta` — Beta — TestFlight / internal track

Updatable and observable: OTA updates wired, crashes and errors reported, basic analytics on the core loop, onboarding for first-run, a way for testers to give feedback.

**Requires capabilities:** `skeleton-shell`, `ota-updates`, `error-reporting`, `analytics`, `onboarding`, `feedback`

**Requires screen categories:** home, settings, profile, onboarding, support

**Checklist:**

- [ ] EAS Update configured (channels, runtimeVersion policy)
- [ ] Crash/error reporting SDK installed and initialized
- [ ] Core-loop events tracked
- [ ] First-run onboarding flow
- [ ] Help/feedback screen reachable from settings
- [ ] eas.json has preview + production build profiles
- [ ] Version/build-number strategy set in app.json *(manual)*

### `store-ready` — Store-ready — App Store / Play review

Everything reviewers and real users hit: permission priming, legal screens, account deletion (Apple requires it when accounts exist), IAP-compliant paywall if monetized, deep links, offline behavior, localization scaffold, icons/splash, and review collateral.

**Requires capabilities:** `skeleton-shell`, `ota-updates`, `error-reporting`, `analytics`, `onboarding`, `feedback`, `permission-priming`, `legal`, `deep-links`, `i18n`

**Requires screen categories:** home, settings, profile, onboarding, support, permissions, auth

**Checklist:**

- [ ] Permission priming screens before every OS permission prompt
- [ ] Privacy policy + terms reachable in-app
- [ ] In-app account deletion (required by Apple when accounts exist)
- [ ] Paywall shows price, term, restore purchases; uses IAP not web checkout
- [ ] URL scheme + universal/app links configured
- [ ] Offline/poor-network behavior defined (banner, cached data, retries)
- [ ] Strings externalized; at least en locale complete
- [ ] App icon, adaptive icon, splash configured in app.json *(manual)*
- [ ] Accessibility pass: labels on touchables, dynamic type doesn't break layouts *(manual)*
- [ ] Screenshots, description, privacy nutrition labels, review notes *(manual)*

### `flagship` — Flagship — native-polish tier beyond store-ready

Everything store-ready demands, plus the native surfaces that make an app feel first-party: haptics on core interactions, home-screen widgets, quick actions, an in-app review loop — and a deliberate decision on Live Activities, watch app, App Intents, share extension, and App Clips for the domains where they fit.

**Requires capabilities:** `skeleton-shell`, `ota-updates`, `error-reporting`, `analytics`, `onboarding`, `feedback`, `permission-priming`, `legal`, `deep-links`, `i18n`, `haptics`, `widgets`, `quick-actions`, `store-review`

**Requires screen categories:** home, settings, profile, onboarding, support, permissions, auth

**Checklist:**

- [ ] At least one widget shipped (fed by the app-group snapshot)
- [ ] Haptics on primary success, destructive confirm, selection
- [ ] Icon quick actions deep-link to top actions
- [ ] In-app review prompt wired to a delight moment
- [ ] Live Activities: shipped for long-running state, or explicitly ruled out for this domain *(manual)*
- [ ] Watch app / complication: shipped or ruled out *(manual)*
- [ ] App Intents (Siri/Shortcuts/interactive widgets): shipped or ruled out *(manual)*
- [ ] Share extension / App Clip: shipped or ruled out *(manual)*
